Overview

Venturous is a random playback manager, which uses media player's command line interface. Currently only Audacious media player is supported, but adding support for other players should be quite simple.

Venturous is built around one feature, which is missing from most (maybe even all) media players: user-defined items for random playback. This feature is especially useful for grouping separate tracks that are actually parts of one composition. For example, parts of classical music compositions (movements) are often distributed as separate tracks - either separate audio files or separate entries in the cue file. It is usually desirable to listen to the movements in succession and in a proper order.

Venturous features a tree-like playlist, which reflects files' and directories' location. Each item (file or directory) in playlist tree can be marked as playable. Items that are marked as playable can be sent to external player's [temporary] playlist. Normal workflow is the following: after external player's playback is finished, Venturous selects next random playable item, sends it to temporary playlist and starts playback. So, effectively, random playback of user-defined items is achieved.

The simplest way to group several audio files in one playable item is to place them in common directory, add it to Venturous playlist, mark it as playable and unmark or remove underlying auto-marked nodes (if any). Another way to group audio files is providing metadata (cue) file, which contains entries for all files in the group, and adding it to playlist tree.

Venturous features playback history, which allows user to identify previously played items and play them again.

Custom actions similar to those in Thunar file manager are available in playlist's and history's context menus. Custom actions allow user to open selected items in external application. For example: open in file manager or in media player, edit text file, move item(s) to trash.

Another nice feature of Venturous is graceful quit. In many popular desktop environments Audacious would not save playback state if it was running while user shut down / restarted / logged out. Venturous does save its state in these cases in most popular desktop environments. Furthermore, since Venturous manages Audacious, it also ensures that Audacious quits in time and thus saves Audacious playback state too.

Venturous saves preferences, playlist, backup playlist and history separately for each user by default. Default Venturous preferences directory location is ~/.config/venturous (here '~' expands to current user's HOME directory). Preferences location can be configured at Venturous compile time (PREFERENCES_DIR CMake cache variable). There is Preferences directory action in Venturous File menu that provides quick access to preferences files in file manager.

Requirements
  1. qt4 OR qt5 shared libraries (Core, XML and GUI/Widgets modules)
  2. Audacious (managed Audacious mode in Venturous - which is not the default mode - requires Audacious 3.4 or later).
  3. audacious and audtool executables must be present in PATH. There are two problems with this requirement in MS Windows:
    1. Audacious bin directory is not added in PATH environment variable automatically. User has to manually add it there.
    2. audtool is usually unavailable in Windows at all. With audtool missing only basic Venturous features work. The biggest problem is that stopping playback does not work in this case, which prevents Venturous from normal functioning.
  4. notify-send from libnotify (optional - for desktop notifications)
  5. Song Change Audacious plugin (optional - for detached Audacious mode, which is the default mode). This plugin is usually unavailable in MS Windows, so Windows users have to switch to the managed mode.

Adding files to playlist

Adding files can be done in playlist edit mode. There are menu actions (Playlist menu) and toolbar buttons that allow adding individual files or recursively add directories. There are also options on toolbar and in preferences that influence adding directories policy.

Venturous adds two types of items in playlist tree: files and media dirs. Add files or Add media dirs checkboxes in Adding directory tab of Preferences must be checked to enable adding these items. Alternatively, adding these items can be enabled/disabled via corresponding toolbar buttons: Consider adding files, Consider adding media dirs.

Naturally, not all files or directories will be added in playlist tree. Only files that match checked File patterns and directories that contain (directly!) files that match Media directory file patterns will be added. These patterns can be configured in Adding directory tab of Preferences. Only metadata patterns are enabled in File patterns by default. This means that only *.cue files will be added as playable items. All audio patterns supported by Audacious are enabled in Media directory file patterns by default. This means that all directories that directly contain audio files will be added as playable items (assuming that Add media dirs is enabled).

A conflict situation is possible with adding directories: when both adding files and media dirs is enabled, and media dir contains files that match File patterns. Adding both directory and its files might be undesirable. This situation is designated as If both.... There are two checkboxes in Adding directory tab of Preferences for fine-tuning adding behavior in this situation. There are also two corresponding buttons on toolbar.

See also Editing playlist tree

Usage example: adding different categories of music

Assume that we have 3 categories of music: individual audio file, directory with interconnected files, large audio file + cue file. How to add these all to Venturous playlist? It can be done as follows:

  1. Separate music that belongs to these categories in 3 directories. Let us name them files, dirs and cue respectively.
  2. Check Add media dirs in Adding directory tab of Preferences (or Consider adding media directories on toolbar). Uncheck Add files in Adding directory tab of Preferences (or Consider adding files on toolbar). Trigger action Add directory and select dirs.
  3. Uncheck Add media dirs and check Add files. Make sure that only default file patterns are checked (all metadata patterns) in Adding directory tab of Preferences. Add directory -> select cue.
  4. Go to Adding directory tab of Preferences. Select any item in File patterns list to transfer focus. Press CTRL+A to select all file patterns. Press Space to invert all file patterns' checked states. Press Escape to close Preferences window (changes are applied automatically). Add directory -> select files.

Playlist tree

Playlist tree is protected from accidental changes. Protection is achieved by introducing two modes: non-edit (default on Venturous start) and edit mode. Playlist's background is greenish in non-edit mode and purplish in edit mode.

Checked items are considered for random playback selection and can be played manually by activating (double click) in non-edit mode. Extended selection is available in both modes (multiple items can be selected with SHIFT or CTRL keys; CTRL+A selects all items). Pressing Enter or Return key starts playing all playable descendants of selected items in both modes.

Apart from obvious folding/unfolding nodes using '+' symbol, there are other ways of changing playlist view. Double clicking playlist entry folds/unfolds it, but be aware that it also activates it in non-edit mode, so if the entry is playable, it will be played. The following keys for manipulating current entry folded state are supported (these are the standard Qt key bindings, so excerpts from Qt documentation follow):

  • Up - moves the cursor to the item in the same column on the previous row. If the parent of the current item has no more rows to navigate to, the cursor moves to the relevant item in the last row of the sibling that precedes the parent.
  • Down - moves the cursor to the item in the same column on the next row. If the parent of the current item has no more rows to navigate to, the cursor moves to the relevant item in the first row of the sibling that follows the parent.
  • Left - hides the children of the current item (if present) by collapsing a branch.
  • Minus - same as Left Arrow.
  • Right - reveals the children of the current item (if present) by expanding a branch.
  • Plus - same as Right Arrow.
  • Asterisk - expands all children of the current item (if present).
  • Page Up - moves the cursor up one page.
  • Page Down - moves the cursor down one page.
  • Home - moves the cursor to an item in the same column of the first row of the first top-level item in the model.
  • End - moves the cursor to an item in the same column of the last row of the last top-level item in the model.

There is a related option in General tab of Preferences: Auto unfolded levels in the tree. It determines how many levels of the tree are unfolded by default.

Editing playlist tree

When Venturous starts, playlist is in non-edit mode. User has to check menu action Edit mode in Playlist menu or toolbar button with the same name in order to enter edit mode.

In edit mode selected items can be removed by pressing Delete key. If an item is removed, all its descendants are also removed recursively. Single item can be checked/unchecked by mouse-clicking. Pressing one of the following keys while playlist has focus will apply the corresponding action to all selected items:

  • Insert - check
  • Backspace - uncheck
  • Space - toggle checked state

When user triggers action that requires edit or non-edit tree mode enabled, and if this mode is not currently active, dialog appears, asking if user wants to switch edit mode.

Adding files and directories is also enabled only in tree edit mode. For more information about adding items to playlist see Adding files to playlist

There are several additional actions in Playlist menu:

  1. Clean up - removes all items that are not playable and have no playable descendants.
  2. Clear - removes all items from playlist.
  3. Restore previous - restores playlist that was used before the last saved playlist.
  4. Load ... - loads previously saved playlist. Note: loading items file from Venturous preferences directory is equivalent to cancelling unsaved playlist changes; loading items.backup file is equivalent to Restore previous action.
  5. Save as ... - saves current playlist to the file selected by user. Playlist tree must be in non-edit mode in order to be saved.

While playlist is being edited, old saved playlist is used for random playback. Trigger Apply changes action in Playlist menu to save changes and start using edited playlist. Trigger Cancel changes in the same menu to cancel editing without saving. Alternatively, unchecking Edit mode in menu or toolbar will show a dialog that asks whether changes have to be saved.

It is also possible to manually edit playlist tree. It is stored in <Venturous preferences directory>/items file in plain text format. Note that tabs should not be replaced by spaces in this file. Only tab character designates tree level indentation. items.backup file in the same directory has identical format.

In case of urgent Venturous quit (such as an urgent quit notice from desktop environment), if playlist is in edit mode, changes are applied and saved.

Managing playback

There are two modes of interaction between Venturous and Audacious: detached (default) and managed.

In the detached mode Venturous stores certain Audacious settings in the Audacious configuration files (these modifications are reverted when the user switches to the managed mode). These automatically set settings ensure that ventool next command is called by the Song Change Audacious plugin when playback of the current Audacious playlist reaches an end. Note that since the detached mode depends on Audacious configuration files, issues are possible if Venturous is launched before these files are created or after Audacious is updated. The workaround to restore settings: open Venturous Preferences window; switch to managed mode; close Venturous Preferences window; open it again, switch to detached mode and close the Preferences window.

In the managed mode Venturous invokes Audacious with the -q command line option and controls the Audacious process. Venturous assumes that the current Audacious playlist reaches an end whenever Audacious process exits. Of course, this is not always the case - Audacious may be closed by the user, the process also exits when the user triggers stop playback action in Audacious. This possibly confusing behavior combined with the constant restarting of Audacious defines problems with the managed mode. Also note that this mode requires Audacious version 3.4 or later (because the -q command line option was introduced in 3.4). But the managed mode is not without advantages: since Venturous has an access to Audacious standard and error output, common issues are detected and the appropriate error messages are shown by Venturous.

There are three playback statuses: stopped, paused and playing. Stopped status means that Audacious is not running or that playback is stopped; paused - playback is paused; playing - playback is active.

There are nine actions, External player and History submenus in Playback menu. All these 9 actions and all 4 actions from the External player submenu are available as ventool commands. Five most common actions and two actions from External player submenu are present on toolbar and in notification area icon context menu.

  1. Play - launches external player (if needed) and continues playing whatever was played the last time in that external player (Audacious).
  2. Pause - pauses playback.
  3. Stop - exits external player.
  4. Previous - plays previous item from history.
  5. Replay last - plays current item from history.
  6. Next from history - plays next item from history.
  7. Next random - plays random item from playlist.
  8. Next - if Next from history option is enabled in preferences, tries Next from history action first. If the option is disabled or there is no next item in history, triggers Next random action.
  9. Play all - plays all playable items from playlist.

Common behavior of the last six actions (all except Play, Pause and Stop):

  • In case of success, external player playback is started with the specified item(s) in a temporary playlist.
  • In case of failure (no corresponding item in history or empty playlist), Stop action is triggered.

External player submenu contains four actions: Show window and Hide window, which allow to show/hide Audacious main window; Set essential options that allows to set necessary external player playback options manually; Update status that allows to query external player playback status manually.

All external player's functionality - both actions and options - is also available to the user. Recommended Audacious options (can be configured in Audacious settings):

  • Playlist -> Continue playback on startup - continues playing track from the saved position, not from the beginning.
  • Plugins -> General -> Status icon - displays Audacious icon in notification area (enable it only if your desktop environment / window manager supports application icons in notification area).

Apart from described above Next from history option, Playback tab of Preferences contains History options and other playback options:

  1. External player - allows user to choose the preferred mode of interaction between Venturous and Audacious.
  2. Always set external player options - this option is on by default, which ensures that external player is configured properly. But if the user desires to change those options manually to achieve some peculiar behavior or if he never changes them and wants to save CPU cycles, switching setting options off can be useful.
  3. Always hide external player window - allows to hide Audacious window each time player is launched. This option is especially useful if Audacious notification area icon (Status Icon) is disabled and the managed mode of interaction is used. In this case hiding external player window prevents Audacious window popping up annoyance. Note that with Audacious Status Icon disabled, user has to issue audtool mainwin-show on command or trigger Show external player window action in Venturous Playback menu / toolbar in order to show Audacious main window after hiding it.
  4. Exit external player on quit - if enabled, external player would be exited when Venturous quits; otherwise it would remain running. This option is available only in the detached mode because the external player must be exited when quitting in the managed mode.
  5. Update status regularly and Status update interval - allow to set the time interval at which Venturous would query Audacious playback status in order to display the correct status to the user in the Venturous window. This option is not important for Venturous functioning, but it may prove convenient for the user. Note that this option increases CPU usage in idle state substantially, especially in case of small time interval, so it is disabled by default.
  6. Desktop notifications - if enabled, desktop notification is shown after played item is changed. If notify-send executable from libnotify is not available in PATH, notifications will not be shown and this option should be turned off to avoid starting useless process.
  7. Startup action - allows triggering one of four playback actions on Venturous start. Note that more sophisticated startup behavior can be achieved with Ventool commands. For example:
    venturous & sleep 1 && ventool previous
    command allows to play previous item from history at the start. See also Ventool and global shortcuts

Usage example: enable searching and playing music in Audacious as an alternative to tree playlist in Venturous

Audacious supports multiple playlists in tabs. So the following can be done:

  1. Add all your music to Venturous playlist in your preferred way (see Adding directories example).
  2. Trigger Play all action.
  3. Rename Audacious temporary playlist Now Playing to All music (playlist tab context menu has Rename ... action).
This way you can use Venturous, which will always add files to Now Playing playlist, and have another permanent Audacious playlist with all your music.

Playback history

Playback history displays previously played items (latest on top). History can be exported to plain text file and imported from file in the same format. These actions are accessible in History submenu of Playback menu. This submenu also contains Clear action, which removes all entries from playback history.

Extended selection of entries is supported. Pressing one of the following keys while History window is active, will apply corresponding action to all selected entries:

  • Enter (or Return) - start playback of selected entries.
  • Delete - remove selected entries.

If history entry is activated (double click), playback of this entry is started. Pressing Backspace key sets current history index to -1 (this index is also set when multiple items are played). The effect of pressing Insert key varies depending on the number of selected entries:

  • If no entries are selected, has no effect.
  • If one entry is selected, it becomes current.
  • If multiple entries are selected, result is equivalent to pressing Backspace key.

History options are available in Playback tab of Preferences:

  • Maximum size - maximum number of entries in history. Big number of history entries (more than several thousands) can cause performance issues, so this value is limited. <Maximum size> latest entries are stored in history.
  • Copy played entry to top - allows to record items, replayed from history, thus providing more accurate playback log. The downside of copying entry to top is that it makes navigating history further than penultimate entry inconvenient (impossible with Previous action).
  • Number of hidden directories - allows to hide first N directories (or, if negative, leave last -N components) of item path in History window and status bar. It is useful because usually first directories are common for all or almost all entries. For example, all items in playlist could start with /home/<user name>/Music/. It is preferable then to hide this common prefix. Note that absolute path is always available in tooltip both for History entries and for status bar label.

Ventool and global shortcuts

Command line tool that controls Venturous is built and installed with Venturous by default (switching off BUILD_VENTOOL CMake option during Venturous build process prevents building and installing it). All available Ventool commands can be found by running ventool help command from terminal.

Ventool allows to manage playback and quit Venturous from command line. It is mandatory for the default detached interaction mode. It can also be used to create various global keyboard shortcuts. One shortcut can be assigned to venturous command (launching Venturous); another one - to ventool quit (quitting Venturous). Ventool playback commands provide global multimedia shortcuts interface. Exact procedure of assigning shortcuts to commands depends on user's desktop environment / window manager (it is usually quite simple).

Notification area icon

There is an option to show Venturous icon in notification area in Venturous preferences. There are also two related suboptions under this option.

Notification area icon is disabled by default because many popular desktop environments and window managers do not support placing application icons in notification area or even do not have notification area at all. If you enable notification area icon, close Preferences window, but still can not see Venturous icon in notification area, it probably means that this feature is not supported in your environment. In this case it is advised to disable notification area icon without delay. Obviously, notification area icon would unjustly consume some small amount of RAM and use some processor time. The bigger problem with inaccessible icon is the following: if at least one of notification area suboptions is enabled, there can be a situation, when Venturous is running, but you have no easy access to its graphical user interface. Suboption Start Venturous to notification area is especially problematic in this case. Fortunately you can still issue commands via Ventool - and there is a Ventool command to show Venturous window.

If you are lucky enough to have notification area icon support in your environment, you get several features. Left or middle click on Venturous notification area icon shows/hides Venturous window. Right click shows a context menu with the following actions:

  • Play
  • Pause
  • Stop
  • Previous
  • Next
  • External player (submenu)
    • Show window
    • Hide window
    • Set essential options
    • Update status
  • Preferences ...
  • Quit

Configuring Venturous

Compile-time configuration (CMake options and cache variables)

  • FORCE_QT4 : BOOLEAN (default: OFF) - if enabled, Qt4 will always be used; otherwise, Qt5 is used if available.
  • PREFERENCES_DIR : STRING (default: ~/.config/venturous) - directory that contains user preferences and playlist (here '~' expands to current user's HOME directory).
  • ALWAYS_EMBED_ICONS : BOOLEAN (default: ON) - if disabled, embedding icons will be done only in MS Windows. It is not recommended to switch embedding off because in this case short text labels instead of some icons will be displayed on toolbar. Saving less than 1 Mb of disk space due to smaller Venturous executable size usually isn't worth the inconvenience. Note: if all necessary icons are present in system theme (which is unlikely), there may be no need to embed icons.
  • ADAPT_CUSTOM_ACTIONS_IF_WIN32 : BOOLEAN (default: ON) - has effect only in MS Windows. If ON, default custom actions and directory separators in paths passed to custom actions are adapted for MS Windows.
  • DEBUG_VENTUROUS : BOOLEAN (default: OFF) - if enabled, details of internal Venturous workflow will be written to program output (stdout). Enabling this option harms performance, so it is disabled by default. This option is useful during Venturous development.
  • BUILD_VENTOOL : BOOLEAN (default: ON) - if enabled, Ventool is built along with Venturous. If you do not need command line interface and global shortcuts for Venturous and you prefer the managed interaction mode, feel free to disable this option. Don't forget to select Audacious (managed) as an External player in the Playback tab of Preferences. See also Ventool and global shortcuts

Run-time configuration (Venturous preferences)

Venturous preferences can be accessed in several ways: Preferences ... action in File menu, corresponding toolbar button or notification area icon's context menu. Each option in Preferences has a verbose tooltip. Hover over control element (spinbox, checkbox or combobox) to activate tooltip.

Preferences window has five tabs:

  1. General - contains many independent option groups. Some of these options are described in Playlist tree and Notification area icon sections.
  2. Playback - contains playback and history options. Most of these options are described in Managing playback and Playback history sections.
  3. Adding directory - contains policy and two editable lists: File patterns, Media directory file patterns. Adding directory policy options and pattern lists are described in verbose tooltips and in Adding files to playlist section. Possible ways of tuning pattern lists follow. Checking/unchecking individual patterns is possible. Extended selection of patterns is supported. You can add new patterns using Add pattern button; edit patterns; copy patterns to another column using Copy right and Copy left buttons. Pressing one of the following keys will apply corresponding action to all selected patterns:
    • Insert - check
    • Backspace - uncheck
    • Space - toggle checked state
    • Delete - remove
  4. Custom actions - allows customizing context menu actions on selected items in playlist and history. Detailed description of custom actions' configuration can be accessed by checking Show help checkbox under the table. Tooltips of buttons under the table describe effects of clicking these buttons. Several custom actions are available by default. These default custom actions are examples and can be edited or removed just as well as actions created by user.
  5. Revert/restore - does not contain any options but allows to revert changes in other tabs. User can either revert changes since opening Preferences window or restore default values in any preceding tab.

There are two options in General and Playback tabs of Preferences to save preferences and history to disk immediately. These options can be especially useful with desktop environments and window managers that do not warn Venturous before quit but just terminate the process. In this case preferences can not be saved automatically before quit, so saving them immediately is desirable.

Additional run-time tweaks

  • Resizing and moving Venturous, History and Preferences windows is possible.
  • Venturous toolbar has a handle that allows to detach and move it. Toolbar can be attached to any window side (top, bottom, left or right). In order to attach toolbar to desired Venturous window side, its handle has to be dragged to the corresponding edge of the window.
  • Venturous toolbar can be hidden. Toolbar action has to be toggled in context menu of main menu bar or toolbar in order to change toolbar's visibility.
  • Identical moving, attaching and hiding of History window is possible. History window has its own title bar, which can be used for moving in place of toolbar handle.
  • There is an option in General tab of Preferences: Status bar. If checked, status bar is displayed with the last played item label on it. If History window is visible, status bar information is redundant. But if History is hidden, status bar might be useful.

All tweaks, described above, are saved to preferences and thus preserved between Venturous launches.

If you want to restore default value of some option, quit Venturous; edit <Venturous preferences directory>/Venturous.xml: remove option's opening tag, closing tag and everything in between. For example, in order to restore Ventool check interval default value, you can do the following:

  1. Quit Venturous.
  2. Remove line <VentoolCheckInterval>1234</VentoolCheckInterval> from <Venturous preferences directory>/Venturous.xml.
  3. Start Venturous. Default value of the option is restored, you can verify this by opening Preferences window.

If you want to restore all default values, you can click Restore defaults button (Target tab = <all>) in Revert/restore tab of Preferences or just remove/rename file Venturous.xml. You can also copy your preferences (as well as playlist and history) to other user or computer by copying file(s) to the corresponding directory. Beware of differences in different Venturous versions though: some options are added, others are renamed or removed. But these differences do not usually cause any issues except that some options are not picked up from preferences file and are set to default values.
